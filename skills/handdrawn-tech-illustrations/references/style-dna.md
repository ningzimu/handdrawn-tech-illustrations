# Style DNA

Use this file to choose and lock the illustration style.

## Friendly Handdrawn Tech

Use for technical article covers, body illustrations, AI/software/product explanations, course-note style diagrams, and commercial publishing.

Core traits:

- Near-white warm paper, around `#FBFAF5` or `#FCFBF7`; not yellow, beige, old paper, or kraft.
- Avoid full-page borders unless requested.
- Friendly handdrawn linework: rounded, slightly playful, still clear enough for technical explanation.
- Soft pencil hatching and small construction marks; keep the drawing handmade rather than polished corporate.
- Article-style technical explainer composition: one clear visual idea, plus a few purposeful annotations, callouts, or micro-panels.
- Clear Chinese handwritten labels, short notes, and small annotation tags.
- Warm pastel marker accents: pale blue, sage green, pale peach, soft lavender, and light yellow when useful.
- Meaningful negative space with medium information density; avoid empty-looking images.

The final image should feel like a friendly technical author drew a useful concept explanation: approachable and slightly cartoon-like, but still structured and readable.

Avoid:

- Generic business infographic.
- Dense brainstorming whiteboard.
- Editorial card layouts.
- SaaS dashboard cards.
- Overly childish cartoon poster.
- Huge typography with a tiny diagram.
- One tiny object floating in mostly empty canvas.
- Gradients, shadows, neon, stickers, decorative blobs.

## Shared Rules

- One image explains one idea.
- Let the source material decide the visual form; do not default to any fixed object family.
- Keep visible Chinese text readable, relevant, and grounded in the article.
- Use 3-6 main labels, note tags, or short phrases whenever the idea needs them.
- Leave meaningful whitespace.
- Do not write the structure type on the image.
- Do not add fake UI text, URLs, English filler, watermarks, or random symbols.

## Canvas Guidance

- 16:9 body illustration: main visual can use 60-75% of width and 45-65% of height, with small supporting notes around it.
- 21:9 cover: one refined visual hook about 50-55% of width.
- 3:4 cover: fuller composition, but still one clear visual hook.
- 1:1 cover: short text and small motif; no long secondary line unless needed.

## Optional Abstraction Prompts

Use these only when the idea is too abstract or the prompt feels generic. They are prompts for thinking, not required steps.

Possible starting questions:

1. What point should the reader understand after looking at the image?
2. Should the image show source evidence, a relationship, a state, a contrast, a scene, or a compact summary?
3. What visible elements are already implied by the article, screenshot, product, or note?
4. Which elements can be omitted so the image does not become a crowded generic explainer?

Avoid repeating the same visual elements across images unless the source material specifically calls for them.

## Rich Explainer Patterns

Use these as flexible starting points, not templates:

- **Main object plus callouts**: one central object, 3-5 small labels, and one small takeaway note.
- **Before / after**: two simple sketch panels with a thin handdrawn arrow and 2-3 small reason labels.
- **Three-step process**: three small objects or stations connected by light arrows, plus one final state.
- **Comparison / tradeoff**: two balanced zones with a clear difference in state, evidence, or consequence.
- **Layered system**: a central object surrounded by 3-5 compact modules, using thin lines and bracket notes.
- **Article margin note**: one larger visual focus with small handwritten notes placed like an author's annotations.

Prefer varied compositions across a multi-image article package. Style consistency means stable paper tone, line weight, palette, and label treatment; it does not mean repeating the same central rack or flow layout.

## Style Sense

Keep the image approachable, slightly cartoon-like, and useful for explaining the source material. Avoid locking the visual into a repeated prop set or a generic template.
